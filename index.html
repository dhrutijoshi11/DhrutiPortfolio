AI Enhanced Resume

The AI-Enhanced Resume Job Evaluator is an innovative web application aimed at empowering job seekers to optimize their resumes for targeted job descriptions. By integrating cutting-edge AI technologies, including Google's Generative AI (Gemini), the platform offers personalized insights and actionable recommendations to enhance resume effectiveness and job relevance.

Key Features:

  • Resume Evaluation: Analyze your resume against specific job descriptions for alignment.

How It Works:

  1. Enter a job description into the application.
  2. Upload your resume (PDF format).
  3. Select the action you wish to perform (e.g., Evaluate Resume, Identify Missing Keywords).
  4. Submit your inputs to receive detailed feedback and actionable insights.

ReviewFlix: Social-Media Powered Movie Recommender with Sentiment and Emotion Analysis

ReviewFlix is an innovative movie recommendation system that enhances traditional recommendation algorithms by incorporating sentiment and emotion analysis from social media and movie review platforms. By analyzing users' sentiments and emotional responses to films, the system offers personalized and contextually relevant movie recommendations, ensuring a more engaging user experience. This project leverages data from IMDb and Rotten Tomatoes to create a highly personalized and accurate recommendation engine based on real user emotions and sentiments.

Key Features:

  • Sentiment Analysis: Using the BERT transformer model, the system classifies movie reviews and tweets from IMDb and Rotten Tomatoes as positive or negative, offering insights into users' emotional reactions to films.
  • Emotion Detection: The system utilizes Convolutional Neural Networks (CNN) and Recurrent Neural Networks (RNN) to identify emotions like "Fabulous," "Satisfy," "Disappointed," and more, based on movie-related text, providing deeper emotional context to recommendations.
  • Personalized Recommendation Engine: The recommendation engine combines sentiment, emotion, and user ratings from IMDb and Rotten Tomatoes to generate highly tailored movie suggestions.
  • User-Friendly Interface: A seamless web interface enables users to input preferences, receive personalized recommendations, and explore movie details, ratings, and reviews.

Technologies used:

  • Python (>=3.6)
  • Libraries: Flask, Pandas, Scikit-learn, TensorFlow/Keras, BERT, NLTK, and more.

Data Collection

Use the "dataset_creation.py" script to scrape movie-related data from IMDb and Rotten Tomatoes. This will generate the final dataset in "dataset.csv".

Sentiment and Emotion Analysis

  • Execute sentiment analysis scripts ("EDA_Bert_Analysis_RT.py" and "EDA_Bert_Analysis_imdb.py") to preprocess, analyze, and apply BERT-based sentiment analysis on movie reviews.
  • After sentiment analysis, two columns—"sentiment_imdb" and "sentiment_rt"—will be added to the dataset, classifying each review as positive or negative.
  • Emotion analysis is performed using the "EmotionAnalysis.py" script, detecting emotions such as "Fabulous," "Satisfy," and "Disappointed" in movie reviews.
  • Post-emotion analysis, the dataset will include two new columns—"Emotion_imdb" and "Emotion_RT"—indicating the detected emotion for each review.
  • All results are saved in "data.csv" and "final_data.csv" within the "sentiment analysis" and "Emotion analysis" folders, respectively.

Recommendation Engine

  • Run the "score_compute.py" script to compute scores for each movie based on sentiment, emotion, and ratings from IMDb and Rotten Tomatoes.
  • The computed scores are saved in a new CSV file, which is used by the recommendation engine.
  • To start the recommendation system, run the "app.py" Flask application, and open your browser at http://localhost:5004/ to interact with the system.

Technical Overview

ReviewFlix uses machine learning techniques like BERT for sentiment analysis and CNN/RNN for emotion detection to gain deeper insights into users' emotional responses to movies. By combining these insights with user ratings, the recommendation engine generates highly personalized movie suggestions that align with users' emotional and sentiment-based preferences. The integration of social media data and review platforms ensures a dynamic and constantly evolving recommendation system, capable of adapting to shifting trends and user sentiments.

Future Enhancements

  • Expand sentiment and emotion analysis to include additional sources like Twitter and user reviews.
  • Improve the recommendation algorithm with collaborative filtering to consider users with similar emotional profiles.
  • Implement a feedback loop where users can rate the recommendations, improving future suggestions based on their interactions.

Crime Rate Prediction Model for Los Angeles

This project involves the development of a predictive model aimed at forecasting crime rates across Los Angeles, helping local authorities enhance public safety and optimize resource allocation. By leveraging historical crime data, the model provides actionable insights into crime trends, enabling proactive measures to address potential crime hotspots. This data-driven approach supports decision-making for law enforcement agencies, fostering safer communities.

Project Objective

The primary objective is to build an accurate predictive model capable of forecasting crime occurrences in Los Angeles, offering valuable insights to law enforcement for strategic resource allocation and targeted interventions.

Data Source

The model leverages publicly available crime data from [Crime Data from 2020 to Present](https://catalog.data.gov/dataset/crime-data-from-2020-to-present), ensuring access to accurate and up-to-date information for analysis and forecasting.

Data Preparation

Preprocessing and Feature Engineering

  • Data cleaning was performed to address missing values, remove duplicates, and standardize data formats for seamless analysis.
  • Date-time features were processed, including extracting day of the week, month, and hour to capture temporal patterns influencing crime rates.
  • Geospatial data was verified to ensure accurate location representation for crime incidents, enabling precise crime hotspot identification.

Exploratory Data Analysis (EDA)

  • Crime Distribution Analysis: The top crime categories were theft, burglary, and violent offenses, with vehicle theft being the most prevalent. This analysis helps in identifying patterns of criminal activity and potential areas for intervention.
  • Crime Hotspot Identification: The highest crime rates were concentrated in Central, 77th Street, Pacific, Southwest, and Hollywood areas. This spatial analysis enables targeted resource deployment in high-crime regions.
  • Gender-based Crime Distribution: Analysis of crime incidents by victim gender revealed no significant differences, but raised questions about the representation of unknown gender data, which warrants further investigation.
  • Seasonal Crime Trends: Crime incidents showed varying patterns over months and time of day. Notably, peak crime times occurred at noon and around dinner time, providing critical data for law enforcement to predict crime spikes.
  • Age Demographics in Crime Incidents: The analysis identified age-related crime patterns, including notable peaks at ages 50 for White and 35 for Unknown, indicating specific age groups may be more susceptible to certain types of crime.
  • Victim Demographics by Area: Latinx, White, and Black communities were found to be most affected by crimes in high-crime areas. Understanding demographic vulnerabilities allows law enforcement to tailor community outreach and intervention strategies.

SARIMAX Model Implementation

The predictive model was built using the SARIMAX (Seasonal AutoRegressive Integrated Moving Average with eXogenous variables) technique. Hyperparameter optimization was performed using grid search, resulting in the model (1, 0, 2)x(2, 0, [1], 12). This model effectively captured temporal and seasonal trends in crime data, offering a reliable forecast. The model achieved an AIC of 520.602 and a BIC of 530.640, reflecting solid fit performance. Further refinement could improve seasonal component significance.

Performance Evaluation

The model's accuracy was evaluated using the RMSE (Root Mean Square Error) metric, yielding a value of 656.30. While the model showed improvement over baseline predictions, there is room for refinement. Future improvements will focus on incorporating additional data sources, refining model features, and continuously optimizing the predictive accuracy for more precise crime forecasting.

Tide Level Prediction

This project evaluates three GRU models for predicting tide time-series data at BHP. Starting with data preprocessing and splitting, each model—simple GRU (Model 1), varied layers and units (Model 2), and different activations (Model 3)—is trained and assessed. Metrics like MSE, RMSE, and MAE guide comparisons, alongside statistical tests and graphical analyses of predictions. Hyperparameter tuning optimizes each model, leading to the selection of the best-performing configuration for accurate tide predictions at BHP.

Technology Used:

  • Python,TensorFlow/Keras (for model building), NumPy (for numerical operations), Pandas (for data manipulation), Matplotlib (for data visualization), Scikit-learn (for evaluation metrics)

Key Features:

  • Model 1: Simple GRU – A basic GRU architecture for time-series forecasting with a single layer of GRU units.
  • Model 2: GRU with Multiple Units and Layers – Experimentation with the number of GRU units and layers to optimize prediction performance.
  • Model 3: GRU with Different Activation Functions – Modification of activation functions (e.g., tanh, sigmoid, ReLU) to improve model performance and accuracy.
  • Time-Series Data Processing – Preprocessing of tide data for sequential predictions using GRU networks.
  • Model Evaluation – Comparison of models to determine the best-performing GRU configuration.

Dataset:

  • Training Data: BHPtraining.csv (Tide data from 2017-2019)
  • Testing Data: BHPtest.csv (Tide data from 2020)

Future Work:

  • Hyperparameter Tuning: Optimizing model parameters such as learning rate and batch size for better accuracy.
  • Advanced Models: Exploring LSTM (Long Short-Term Memory) networks or hybrid models for more robust predictions.
  • Real-Time Prediction: Extending the model to predict tides in real-time with granular data input.
